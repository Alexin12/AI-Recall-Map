# AI-enriched Concept format is a fixed six-field template

Each Concept is extracted into a fixed template — `keyword`, `analogy`, `technical_explanation`,
`code_snippet` (optional), `core_claim` (optional), `source_excerpt` — instead of the single
`explanation` blob used through M2. Every field is extracted from the Material when the Material
already contains it, and generated by the model only when it doesn't, **except `code_snippet`,
which is extract-only and never fabricated**: absent code in the Material means the literal
string `"none"`, never invented or completed code. Any field the model had to generate rather than
find is flagged in `ai_supplemented_fields` on the contract and shown as an "AI-supplemented"
badge in the UI, so the learner can tell what came from their own notes versus what the model
added. `source_excerpt` always anchors the Material span the Concept came from, and Concept
detail keeps it behind a `View source` toggle rather than showing it inline by default.

`keyword` and `source_excerpt` are aliases of the existing `name` and `source_snippet` columns —
no new storage for those two. `analogy`, `technical_explanation`, `code_snippet`, `core_claim`,
and `ai_supplemented_fields` are new `concepts` columns, added via a single idempotent migration.
The extraction prompt (`app/prompts/extraction_v1.py`) is extended to ask for all five in the
same LLM call that already produces `explanation`, so the enriched fields cost no extra
round-trip.

**Status**: accepted

**Consequences**:
- `Concept` (backend/app/extraction.py) and `ConceptDetail` (backend/app/concepts.py) carry the
  six fields; `insert_concept` and `list_concepts` read/write the four new columns.
- The extraction prompt's regression coverage (`test_enriched_concept.py`, including the reserved
  FastAPI eval sample) must keep asserting `code_snippet` is never fabricated.
- The existing `explanation` field is untouched and still drives review-question generation; the
  six-field template is additive, not a replacement of the review pipeline.
